Degradados

linear-gradient

Un degradado lineal es la forma más habitual de degradado (de un punto a otro siguiendo una línea recta y con dos o más colores). Todos los degradados se tratan como imágenes de fondo, y se declaran en la propiedad background-image o background.

La forma de escribirnos es declarando el ángulo (en grados) o la dirección y los diferentes colores de corte.

Ejemplo de degradado entre dos colores de arriba abajo:
div{background-image: linear-gradient(180deg, yellow, blue);}

Ejemplo de degradado entre varios colores, en diagonal:
div{background-image: linear-gradient(to bottom right, yellow 0%, blue 10%, red 50%, purple 100%);}

radial-gradient

Un degradado radial o en forma cicular se declara indicando el tamaño (si queremos), forma (circle o ellipse), posición del vértice, cortes de color.

Ejemplo de degradado circular posicionado en la esquina superior derecha entre dos colores:
background-image: radial-gradient(circle at right top, yellow, blue);

Ejemplo de degradado elíptico centrado entre 4 colores:
background-image: radial-gradient(ellipse at center, yellow 10%, blue 20%, green 50%, brown 100%);

repeating-linear-gradient

Este tipo de degradado nos permite crear un patrón o degradado que se repite de forma continua. Ejemplo: repeating-linear-gradient(180deg, #1ac6cc, #1ac6cc 7%, #646464 10%).

La forma de que tenga espacio para repetirse es no declarar el 100% del espacio que ocuparán los colores (en el ejemplo anterior, tenemos hasta el 10%), o bien usar unidades absolutas (píxeles).

Sombras

box-shadow

La forma de declarar sombras en un elemento es con la propiedad box-shadow. Los valores que se introducen son el desplazamiento de la sombra en el eje x, el desplazamiento en el eje y, la difuminación de la sombra, el spread (un espacio de color sólido antes de que empiece el difuminado; es opcional) y el color de la sombra. Además, si usamos la propiedad inset, la sombra se renderizará por dentro del elemento en vez de por fuera.

Ejemplo de sombra normal:
box-shadow: 5px 5px 3px rgba(0, 0, 0, .4);

Ejemplo de sombra con spread:
box-shadow: 5px 5px 3px 2px rgba(0, 0, 0, .4);

Ejemplo de sombra con inset:
box-shadow: inset 5px 5px 3px rgba(0, 0, 0, .4);

Un elemento puede tener más de una sombra, que se posicionan una debajo de la otra. Ejemplo:
box-shadow: 0 5px 0 lightblue, 0 10px 0 hotpink;

text-shadow

La forma de declarar una sombra de texto es igual que la de box-shadow, con la única diferencia de que en text-shadow no existe el inset (no se puede renderizar la sombra por dentro del texto).

Ejemplo: text-shadow: 2px 2px 3px rgba(0, 0, 0, .4);

Transiciones

transition
Para añadir una transición a un elemento en un cambio de estado (por ejemplo, :hover o cuando se le añade o quita una clase), usaremos transition, declarando el tiempo que durará la transición en segundos o milisegundos y la propiedad a transicionar (por defecto, all).
Ejemplo: a{transition: .4s background-color;}

Transformaciones

transform
Permite modificar el aspecto o posición de un elemento sin alterar su entorno (por ejemplo, hacer algo más grande sin mover los elementos de alrededor). Los tipos de transformación habituales son translate(), scale(), rotate(), skewX() y skewY().
transform: translate()
Desplaza a un elemento en el eje X y el eje Y. Ejemplo: 
  .ejemplo{
transform: translate(10px, 20px);
}
transform: scale()
Altera el tamaño de un elemento en base a 1. Ejemplo: 
  .ejemplo{
transform: scale(1.5);
}
transform: rotate()
Gira un elemento a partir de su centro, expresado en grados (deg). Ejemplo: 
  .ejemplo{
transform: rotate(15deg);
}
transform: skewX()
Deforma un elemento sobre el eje X, expresado en grados (deg). Ejemplo: 
  .ejemplo{
transform: skewX(15deg);
}
transform: skewY()
Deforma un elemento sobre el eje Y, expresado en grados (deg). Ejemplo: 
  .ejemplo{
transform: SkewY(15deg);
}

Animaciones

Animaciones CSS

Las animaciones permiten hacer movimientos y transiciones sencillas en elementos web, sin necesidad de interacción por parte del usuario.

Las animaciones se definen con @keyframes y se asignan posteriormente a los elementos que nos interese animar.

Algunos navegadores pueden necesitar todavía que declares las animaciones con su prefijo (-webkit, -moz, etc). Compruébalo en Can I Use.

@keyframes

Define los pasos que seguirá una animación, y le da un nombre para poderse referir a ella más adelante.

Los estilos de cada paso de la animación se engloban entre {}. Los pasos pueden indicarse con from y to, si son solo dos, o con %.

Ejemplo: 

  @keyframes ejemplo{
0%{
  width: 20px;
}
20%{
  width: 50px;
}
80%{
  width: 90px;
}
100%{
  width: 240px;
}
}

animation

Esta propiedad permite asignar una animación a un elemento, indicando en este orden: nombre de la animación, duración en segundos (s) o milisegundos (ms), velocidad (linear, ease-in, ease-out, ease-in-out), retraso en empezar, repeticiones, dirección en la que se produce (normal, reverse, o alternate si queremos que cambie con cada repetición). Tienes algunas opciones más en la Mozilla Developer Network.

Un ejemplo de asignación de animación: 

  .elemento{
animation: ejemplo 3s ease-out 3s 3 alternate;
}
