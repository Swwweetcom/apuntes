Elementos básicos

Comentarios HTML
Los comentarios HTML se escriben entre <!-- y -->, y permiten poner notas para el desarrollador u ocultar partes del código al navegador. Ejemplo: <!-- Contenido principal -->
<!DOCTYPE html>
Define el tipo de documento. Señala que un documento sigue los estándares W3C.
<html>
Etiqueta que engloba todo el documento HTML.
<head>
Etiqueta que engloba la metainformación del documento: título, CSS, JavaScript…
<meta charset>
Etiqueta, dentro del <head>, que define la metainformación del sistema de codificación de caracteres, que suele ser UTF8: <meta charset="utf-8" />.
<meta name="viewport">

Esta etiqueta se pone al principio de nuestro html (en el tag head) para que los navegadores móviles sepan que una web se adaptará correctamente a su tamaño (sea web móvil o o web responsive), y que muestren la página contando con la anchura del dispositivo. La forma correcta de escribirlo sería: <meta name="viewport" content="width=device-width, initial-scale=1.0" />.

En esta etiqueta podríamos hacer otras cosas, como bloquear el zoom al usuario (user-scalable: no dentro del atributo content), pero no es una práctica aconsejable ya que se crea un problema de accesibilidad que casi nunca estará justificado.

<title>
Etiqueta, dentro del <head>, que define el título del documento.
<style>
Etiqueta para añadir estilos CSS dentro de un documento HTML.
<link>
Sirve para añadir un recurso externo, normalmente una hoja de estilos: <link rel="stylesheet" href="styles.css" />
<body>
Etiqueta que engloba el cuerpo (contenido) del documento.

Atributos

id
Identifica un elemento concreto para poder aplicarle estilos o funciones de JavaScript. Cada id es único, solo puede aparecer una vez en cada documento HTML.
class
Identifica a una serie de elementos para poder aplicarles estilos.
href
Especifica la URL de una página, documento, o ancla a la que apunta un enlace.

Elementos de línea

<span>
Engloba una sección, generalmente dentro de un bloque de texto. No tiene un sentido semántico y se suele añadir para poder dar estilos concretos a dicha sección.
<strong>
Señala una porción de texto más importante. Suele representarse con la tipografía en negrita.
<em>
Señala una porción de texto que se quiere enfatizar. Suele representarse con la tipografía en cursiva.
<a>
Marca un enlace hipertextual. El atributo href indica dónde lleva ese enlace: a una página externa, a una sección del documento, a un email...
<i>
Marca una porción de texto en cursiva que no tiene significado semántico ni ninguna importancia especial.
<b>
Marca una porción de texto en negrita que no tiene significado semántico ni ninguna importancia especial.
<br>
Crea un salto de línea en el texto. No se cierra, se puede escribir como <br> o <br />

Elementos de bloque

<div>
Marca un bloque de contenido en el documento. No tiene significado semántico.
<ul>
Marca una lista de elementos no ordenados.
<ol>
Marca una lista de elementos ordenados.
<li>
Marca un elemento de una lista, tanto de una ul como de una ol.

Elementos de texto

<h1>
Marca un encabezado de primer nivel. Solo puede haber uno en cada página.
<h2>
Marca un encabezado de segundo nivel.
<h3>
Marca un encabezado de tercer nivel.
<h4>
Marca un encabezado de cuarto nivel.
<h5>
Marca un encabezado de quinto nivel.
<h6>
Marca un encabezado de sexto nivel.
<p>
Marca un párrafo.

Formularios

<form>
Marca un formulario y engloba todos sus campos y botones.

El atributo action indica la acción que se llevará a cabo cuando se envíe el formulario, clicando el botón de submit (normalmente, la acción consiste en enviar el formulario a un servidor).

El atributo method especifica el método HTTP (GET o POST) que se utilizará al enviar el formulario.

<fieldset>
Marca un conjunto de elementos relacionados dentro de un formulario.
<legend>
Sirve como título o leyenda para un <fieldset>.
<label>
Marca la etiqueta de un campo del formulario. Cada <label> está asociada a un <input>.
<input>
Marca un campo de un formulario, donde el usuario puede ejecutar alguna acción, como escribir. El atributo type define de qué tipo de campo se trata. Algunos de los tipos que existen son:
  • <input type="text" />: Marca un campo de texto libre de una línea.
  • <input type="email" />: Marca un campo de e-mail.
  • <input type="submit" />: Marca un botón con el que validar el formulario.
  • <input type="password" />: Marca un campo de contraseña.
  • <input type="email" />: Marca un campo de e-mail.
  • <input type="tel" />: Marca un campo de teléfono.
  • <input type="date" />: Marca un campo donde introducir una fecha.
  • <input type="number" />: Marca un campo numérico.
  • <input type="search" />: Marca un campo de búsqueda.
  • <input type="radio" />: Marca un botón de opción. Los botones de opción permiten al usuario elegir una única opción de entre las posibles.
  • <input type="checkbox" />: Marca un campo de casilla. Las casillas permiten al usuario elegir cero o más opciones de entre las presentes.

El atributo placeholder permite añadir un pequeño texto de ayuda dentro de un input. Este texto de apoyo desaparece en cuanto el usuario empieza a escribir dentro del campo.

El atributo required indica que el campo en cuestión es obligatorio. Si el usuario lo deja vacío, o no lo rellena de manera adecuada, el navegador dará algún tipo de mensaje de error.

<button>
Marca un botón clicable. Con el atributo type="submit" envía el formulario.
<textarea>
Marca un campo de texto de múltiples líneas, a diferencia del <input type="text" />.
<select>
Marca una lista desplegable de opciones, definidas con el elemento <option>.
<option>
Marca cada una de las opciones dentro de un <select>.

Elementos multimedia

<img>
Marca una imagen. No se cierra, se puede escribir como <img> o <img />. El atributo src indica la ruta a la imagen en cuestión. El atributo alt debe contener una descripción breve de la imagen que servirá como texto alternativo en caso de que falle la carga de la imagen. Ejemplo: <img src="kitten.jpg" alt="Un gatito" />
<video>

Permite usar un reproductor de vídeo nativo. Aunque se podría usar como el tag img, vinculándolo con el atributo src, lo normal es utilizar los tags <source>, poner varios formatos de vídeo diferentes (al menos .mp4 y .webm, con su type indicando el formato), y que el navegador escoja el formato que pueda reproducir.

Ejemplo: 

  <video controls>
<source src="pelicula.webm" type="video/webm">
<source src="pelicula.mp4" type="video/mp4">
<source src="pelicula.ogg" type="video/ogg">
</video>

El atributo controls permite que se rendericen los controles de reproducción para que el usuario pueda controlar el vídeo. Otros atributos interesantes son width y height (para darle un tamaño de reproducción al vídeo), poster="ruta/de/imagen.jpg" (para cargar una imagen de carátula), mute (el vídeo comienza sin sonido), loop (el vídeo se reproduce contínuamente), preload (el vídeo se empieza a descargar aunque el usuario no haya hecho nada) y autoplay (el vídeo empieza a reproducirse en cuanto carga la página).

<audio>
El reproductor de audio nativo comparte casi todas las características con <video>. Los formatos de audio que se emplean son .mp3 (type="audio/mp3") y .ogg (type="audio/ogg").
<iframe>

Un iframe permite incrustar una web externa dentro de nuestra web, en un espacio definido por el width y el height que le demos. Es la forma más habitual de incorporar vídeos de un servicio externo, como Youtube.

Ejemplo de un vídeo con iframe de Youtube: 

  <iframe width="560" height="315"
src="https://www.youtube.com/embed/-qTIGg3I5y8"
frameborder="0" allowfullscreen></iframe>

Este código está utilizando frameborder, un atributo html anticuado para evitar que el navegador pinte un borde al iframe. Opcionalmente, podemos quitar este atributo y poner en nuestra CSS iframe{border:0;}

El atributo allowfullscreen permite que el vídeo se pueda reproducir en pantalla completa.

Cuando introducimos vídeos con un iframe en una web responsive, mantener las proporciones de un vídeo en todos los tamaños de pantalla es complejo. Podemos ayudarnos de herramientas como Embed Responsively, que nos dan una serie de estilos ya preparados y juegan con paddings y porcentajes para simular el aspect-ratio de los vídeos.
