Elementos básicos

Comentarios HTML
Los comentarios HTML se escriben entre <!-- y -->, y permiten poner notas para el desarrollador u ocultar partes del código al navegador. Ejemplo: <!-- Contenido principal -->
<!DOCTYPE html>
Define el tipo de documento. Señala que un documento sigue los estándares W3C.
<html>
Etiqueta que engloba todo el documento HTML.
<head>
Etiqueta que engloba la metainformación del documento: título, CSS, JavaScript…
<meta charset>
Etiqueta, dentro del <head>, que define la metainformación del sistema de codificación de caracteres, que suele ser UTF8: <meta charset="utf-8" />.
<meta name="viewport">

Esta etiqueta se pone al principio de nuestro html (en el tag head) para que los navegadores móviles sepan que una web se adaptará correctamente a su tamaño (sea web móvil o o web responsive), y que muestren la página contando con la anchura del dispositivo. La forma correcta de escribirlo sería: <meta name="viewport" content="width=device-width, initial-scale=1.0" />.

En esta etiqueta podríamos hacer otras cosas, como bloquear el zoom al usuario (user-scalable: no dentro del atributo content), pero no es una práctica aconsejable ya que se crea un problema de accesibilidad que casi nunca estará justificado.

<title>
Etiqueta, dentro del <head>, que define el título del documento.
<style>
Etiqueta para añadir estilos CSS dentro de un documento HTML.
<link>
Sirve para añadir un recurso externo, normalmente una hoja de estilos: <link rel="stylesheet" href="styles.css" />
<body>
Etiqueta que engloba el cuerpo (contenido) del documento.

Atributos

id
Identifica un elemento concreto para poder aplicarle estilos o funciones de JavaScript. Cada id es único, solo puede aparecer una vez en cada documento HTML.
class
Identifica a una serie de elementos para poder aplicarles estilos.
href
Especifica la URL de una página, documento, o ancla a la que apunta un enlace.

Elementos de línea

<span>
Engloba una sección, generalmente dentro de un bloque de texto. No tiene un sentido semántico y se suele añadir para poder dar estilos concretos a dicha sección.
<strong>
Señala una porción de texto más importante. Suele representarse con la tipografía en negrita.
<em>
Señala una porción de texto que se quiere enfatizar. Suele representarse con la tipografía en cursiva.
<a>
Marca un enlace hipertextual. El atributo href indica dónde lleva ese enlace: a una página externa, a una sección del documento, a un email...
<i>
Marca una porción de texto en cursiva que no tiene significado semántico ni ninguna importancia especial.
<b>
Marca una porción de texto en negrita que no tiene significado semántico ni ninguna importancia especial.
<br>
Crea un salto de línea en el texto. No se cierra, se puede escribir como <br> o <br />
<img>
Marca una imagen. No se cierra, se puede escribir como <img> o <img />. El atributo src indica la ruta a la imagen en cuestión. El atributo alt debe contener una descripción breve de la imagen que servirá como texto alternativo en caso de que falle la carga de la imagen. Ejemplo: <img src="kitten.jpg" alt="Un gatito" />

Elementos de bloque

<div>
Marca un bloque de contenido en el documento. No tiene significado semántico.
<ul>
Marca una lista de elementos no ordenados.
<ol>
Marca una lista de elementos ordenados.
<li>
Marca un elemento de una lista, tanto de una ul como de una ol.

Elementos de texto

<h1>
Marca un encabezado de primer nivel. Solo puede haber uno en cada página.
<h2>
Marca un encabezado de segundo nivel.
<h3>
Marca un encabezado de tercer nivel.
<h4>
Marca un encabezado de cuarto nivel.
<h5>
Marca un encabezado de quinto nivel.
<h6>
Marca un encabezado de sexto nivel.
<p>
Marca un párrafo.
