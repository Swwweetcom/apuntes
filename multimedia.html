Elementos multimedia

<video>

Permite usar un reproductor de vídeo nativo. Aunque se podría usar como el tag img, vinculándolo con el atributo src, lo normal es utilizar los tags <source>, poner varios formatos de vídeo diferentes (al menos .mp4 y .webm, con su type indicando el formato), y que el navegador escoja el formato que pueda reproducir.

Ejemplo: 

  <video controls>
<source src="pelicula.webm" type="video/webm">
<source src="pelicula.mp4" type="video/mp4">
<source src="pelicula.ogg" type="video/ogg">
</video>

El atributo controls permite que se rendericen los controles de reproducción para que el usuario pueda controlar el vídeo. Otros atributos interesantes son width y height (para darle un tamaño de reproducción al vídeo), poster="ruta/de/imagen.jpg" (para cargar una imagen de carátula), mute (el vídeo comienza sin sonido), loop (el vídeo se reproduce contínuamente), preload (el vídeo se empieza a descargar aunque el usuario no haya hecho nada) y autoplay (el vídeo empieza a reproducirse en cuanto carga la página).

<audio>
El reproductor de audio nativo comparte casi todas las características con <video>. Los formatos de audio que se emplean son .mp3 (type="audio/mp3") y .ogg (type="audio/ogg").
<iframe>

Un iframe permite incrustar una web externa dentro de nuestra web, en un espacio definido por el width y el height que le demos. Es la forma más habitual de incorporar vídeos de un servicio externo, como Youtube.

Ejemplo de un vídeo con iframe de Youtube: 

  <iframe width="560" height="315"
src="https://www.youtube.com/embed/-qTIGg3I5y8"
frameborder="0" allowfullscreen></iframe>

Este código está utilizando frameborder, un atributo html anticuado para evitar que el navegador pinte un borde al iframe. Opcionalmente, podemos quitar este atributo y poner en nuestra CSS iframe{border:0;}

El atributo allowfullscreen permite que el vídeo se pueda reproducir en pantalla completa.

Cuando introducimos vídeos con un iframe en una web responsive, mantener las proporciones de un vídeo en todos los tamaños de pantalla es complejo. Podemos ayudarnos de herramientas como Embed Responsively, que nos dan una serie de estilos ya preparados y juegan con paddings y porcentajes para simular el aspect-ratio de los vídeos.
